Git vs. GitHub — Core Distinction

What is the fundamental difference between Git and GitHub?

Git is a free, open-source, distributed version control tool that runs on your local machine.

GitHub is a cloud-based platform (owned by Microsoft) that hosts Git repositories and adds collaboration features on top — such as pull requests, issues, and CI/CD pipelines.

The analogy: Git is to GitHub as email is to Gmail.

What Git Does Not Include

What features does Git NOT have on its own?

Git alone has none of the following: - A web interface or dashboard - Pull/merge requests - Issue tracking - CI/CD pipelines - User accounts or access control (beyond filesystem permissions) - Notifications or code review tools

These are all features added by hosting platforms like GitHub or GitLab.

GitHub Key Features

What are five key features that GitHub adds on top of Git?

  1. Pull Requests (PRs) — formal code review and merge proposals
  2. Issues — built-in bug tracker and feature request system
  3. GitHub Actions — CI/CD automation platform
  4. GitHub Pages — free static website hosting from a repo
  5. Organizations & Teams — access control and team management

GitHub is owned by Microsoft and available at github.com.

GitLab vs. GitHub — Key Difference

What is the most significant practical difference between GitLab and GitHub?

GitLab can be self-hosted — the entire platform can be installed on private infrastructure. This makes it the preferred choice for enterprises with data sovereignty requirements or air-gapped environments.

GitHub does not offer a comparable self-hosted option for most organizations (GitHub Enterprise is available but expensive and limited).

GitLab also uses Merge Requests (MRs) instead of Pull Requests, and its built-in CI/CD is often considered more deeply integrated than GitHub Actions.

Pull Requests Are a Platform Feature

Are pull requests a Git feature? Explain.

No. Pull requests (or merge requests on GitLab) are a platform feature — they exist in GitHub, GitLab, and similar services, but not in Git itself.

Git's built-in mechanisms are git merge, git fetch, and git pull. The concept of a "pull request" as a formalized, reviewable, discussable code change proposal was introduced by hosting platforms to facilitate team collaboration.

Git and GitHub Working Together

How do Git (local) and GitHub (remote) work together in a typical workflow?

Step | Tool   | Action
-----|--------|------------------------------------------
1    | Git    | Work locally: commit, branch, stage
2    | Git    | Push local repo to GitHub remote
3    | GitHub | Open pull request, review, run CI/CD
4    | Git    | Teammates pull changes to their local repos

GitHub is a hosted remote with collaboration tooling. You could replace it with any server that accepts Git pushes.

Is GitHub Required to Use Git?

Do you need GitHub (or any platform) to use Git?

No. Git is completely independent of any platform. You can: - Use Git entirely on your local machine with no internet connection - Host a bare Git repo on any server you control - Use Git across a team via SSH without any third-party service

GitHub, GitLab, and others are optional conveniences that add web-based collaboration on top of Git.

Git and GitHub Open Source Status

Which of Git, GitHub, and GitLab are open source?

  • Git — fully open source (GPL license), maintained by the Git community
  • GitHub — proprietary (owned by Microsoft); not open source
  • GitLab — open source (Community Edition is MIT-licensed); an Enterprise Edition with additional features is also available

Deleting a GitHub Repo

If you delete your GitHub repository, do you lose your code?

No — your local Git repository still exists on your machine. GitHub only hosts a copy (remote) of your repository. Deleting the remote on GitHub does not affect your local .git directory or any other clones of the repository that exist elsewhere.
