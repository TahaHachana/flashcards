Verify Git Installation

What command verifies that Git is installed and shows its version?

git --version
# Example output: git version 2.47.0

Install Git on Ubuntu/Debian

What commands install Git on a Debian/Ubuntu Linux system?

sudo apt update
sudo apt install git

Install Git on macOS with Homebrew

What is the recommended way to install the latest Git on macOS using a package manager?

brew install git

Homebrew installs the latest upstream Git, independent of Apple's bundled version. Alternatively, running git --version on macOS will prompt you to install the Xcode Command Line Tools if Git is not already present.

Three Git Config Levels

What are the three levels of Git configuration, and what is their priority order?

Level    | Scope              | File Location          | Flag
---------|-------------------|------------------------|----------
System   | All users          | /etc/gitconfig         | --system
Global   | Current user       | ~/.gitconfig           | --global
Local    | Current repo only  | .git/config            | --local

Priority (highest to lowest): Local → Global → System Local settings always win over global and system settings.

Set Global User Identity

What two commands must you run immediately after installing Git to identify yourself?

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

These are written to ~/.gitconfig and apply to every repository on the machine. This information is embedded in every commit you make as permanent metadata.

Why Identity Configuration Is Required

Why is configuring user.name and user.email the first thing you must do after installing Git?

Git embeds the author's name and email into every commit as permanent, immutable metadata. Without this configuration: - Git may use incorrect system defaults - Some Git versions will refuse to make a commit - Collaborators cannot identify who made which changes

Every commit stores: the snapshot, the author identity, a timestamp, and a commit message.

Per-Repository Identity Override

How do you set a different email address for a specific Git repository (e.g., a work project)?

Navigate into the repository, then use --local instead of --global:

cd /path/to/work-repo
git config --local user.email "name@company.com"

This writes to .git/config inside that repo only, overriding the global setting for that project.

Set Default Editor

How do you configure VS Code as Git's default commit message editor?

git config --global core.editor "code --wait"

The --wait flag tells Git to pause until the VS Code tab is closed before continuing. Without it, Git would continue immediately, reading an empty message.

Other examples:

git config --global core.editor "nano"
git config --global core.editor "vim"

Set Default Branch Name

What command sets the default branch name for all new Git repositories to main?

git config --global init.defaultBranch main

Historically Git used master as the default. The current convention (and default on GitHub and GitLab) is main. This config ensures consistency when you run git init.

View All Git Configuration

What command lists all active Git configuration settings along with the file each comes from?

git config --list --show-origin

To list settings without the origin file:

git config --list

To check a single specific value:

git config user.name
git config user.email

Global Config File Location

Where is the global Git configuration file stored, and what does a typical one look like?

Located at ~/.gitconfig (or ~/.config/git/config). A typical file after basic setup:

[user]
    name = Jane Doe
    email = jane@example.com
[core]
    editor = code --wait
    autocrlf = input
[init]
    defaultBranch = main

Line Ending Configuration

How should you configure core.autocrlf on Windows vs. macOS/Linux, and why?

OS            | Setting | Behavior
--------------|---------|-------------------------------------------
Windows       | true    | LF → CRLF on checkout; CRLF → LF on commit
macOS / Linux | input   | CRLF → LF on commit; no change on checkout

This prevents line ending conflicts in cross-platform teams where Windows uses CRLF and Unix systems use LF.

# Windows
git config --global core.autocrlf true

# macOS / Linux
git config --global core.autocrlf input

Git Built-in Help

What are two ways to get help documentation for a Git command, such as config?

# Full manual page (verbose)
git help config

# Short flag summary (concise, great for quick reference)
git config -h

The -h flag works on any Git command and prints a compact list of available options directly in the terminal.

Post-Install Setup Checklist

What are the four essential configuration steps to run immediately after installing Git?

# 1. Set your name
git config --global user.name "Your Name"

# 2. Set your email
git config --global user.email "you@example.com"

# 3. Set your preferred editor
git config --global core.editor "code --wait"

# 4. Set the default branch name
git config --global init.defaultBranch main

Verify everything with: git config --list
