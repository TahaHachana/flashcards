Buffer Vs Window Distinction

What is the difference between a buffer and a window in Vim?

A buffer is the in-memory container for a file's content (one buffer per file, with a unique unchanging number). A window is a viewport that displays a buffer on screen. The same buffer can appear in multiple windows, and a buffer can exist with no window (hidden).

Listing Buffers

Which three interchangeable commands list all current buffers?

:ls, :files, and :buffers. Append ! (e.g. :ls!) to also show unlisted buffers such as the help buffer.

Buffer Number Permanence

Cloze: Each buffer has a unique, {{nonchanging}} number that stays the same for the whole session.

nonchanging

Buffer Status Flags

What do the buffer status flags shown by :ls mean?

Flag   Meaning
-----  --------------------------------------------------
u      Unlisted buffer (shown only with !)
%      Buffer in the current window
#      Alternate buffer (switch with :e #)
a      Active (loaded and visible)
h      Hidden (loaded but not visible)
-      modifiable off (read-only)
=      Read-only and cannot be made modifiable
+      Modified (unsaved changes)
x      Read errors

Switch To Buffer By Number

How do you switch to buffer number N?

:buffer N (abbreviated :b N). You can also specify the buffer by filename.

Next And Previous Buffer

Which commands move to the next and previous buffers in the list?

:bnext (:bn) moves to the next buffer; :bprevious (:bp, also :bNext) moves to the previous buffer.

Split Variants Of Buffer Commands

What does the s prefix add to buffer commands like :sbuffer or :sbnext?

It opens the target buffer in a new split window instead of replacing the buffer in the current window.

Hidden Buffers

What is a hidden buffer, and what enables hiding instead of forcing a save?

A hidden buffer is loaded in memory but not displayed in any window. Setting the hidden option lets you :edit another file while keeping the current modified buffer in memory (flagged h in :ls) instead of being forced to write it first.

Alternate Buffer Symbol

Cloze: In :ls, the {{#}} symbol marks the alternate buffer, which you switch to with :e #.

Special Buffers

Name four kinds of special buffers Vim uses for its own purposes.

directory (lists files in a directory), help (holds help files loaded by :help), QuickFix (holds compile errors or a location list), and scratch (expendable general-purpose text).

Windo Command

What does :windo cmd do?

It runs the ex command cmd in each window of the current tab, stopping at the first window where the command produces an error.

Bufdo Command

What does :bufdo cmd do, and how does it differ from :windo?

:bufdo cmd runs cmd on every buffer in the session (not just visible ones), stopping at the first error. :windo only operates on visible windows in the current tab.

Windo Substitution Example

Write a :windo command that fixes every occurrence of myPoorlyCapitalizedClass to MyPoorlyCapitalizedClass across all open windows.

:windo %s/myPoorlyCapitalizedClass/MyPoorlyCapitalizedClass/g

Bunload Vs Bdelete

What is the difference between :bunload and :bdelete?

:bunload removes the buffer from memory but keeps it in the buffer list (so it can be reloaded). :bdelete unloads it AND removes it from the buffer list entirely. Add ! to force either on a modified buffer.

Three Windows Three Buffers Setup

Starting Vim on file1, what commands give you three windows showing three different files?

:split file2 followed by :split file3. This produces three windows backed by three buffers (file1=1, file2=2, file3=3).
