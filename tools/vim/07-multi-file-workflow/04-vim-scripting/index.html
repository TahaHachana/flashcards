Already Scripting

Why does having a .vimrc mean you are already writing Vim script?

Every Vim command and option is valid script input, and Vim script provides the usual programming constructs (variables, if/while, functions). A config file is just a script that runs at startup.

Simplest Color Scheme Script

What is the simplest possible color-scheme script you can put in a config file?

A single command, e.g. :colorscheme desert, which applies that scheme on every launch.

If Block Syntax

What is the syntax of a Vim if block, and which parts are optional?

if cond
  " code
elseif other_cond
  " code
else
  " code
endif

The elseif (repeatable) and else blocks are optional; the block always ends with endif.

Ternary Form

What is Vim's terse C-like conditional form?

cond ? expr1 : expr2 — if cond is true, expr1 runs; otherwise expr2 runs.

Strftime Hour

Which built-in function returns the current hour, and how is it called?

strftime() with the format %H, written strftime("%H"). Its optional second argument is epoch seconds and defaults to the current time.

Assigning Variables

How do you assign a value to a variable in Vim script?

With the :let command, e.g. let var = "value".

Default Variable Scope

What is a variable's default scope outside versus inside a function?

Global (g:) when defined outside a function, and local (l:) when defined inside a function.

Why Execute Is Needed

Why does colorscheme colorScheme fail when colorScheme is a variable?

The colorscheme command expects a literal scheme name, not a variable, so it tries to load a scheme literally named "colorScheme" and errors. You need execute to evaluate the variable first.

Execute Command Usage

How do you use execute to run a command built from a variable?

execute "colorscheme " . colorScheme — quoted text stays literal (note the trailing space), and the bare variable colorScheme is evaluated and substituted.

Defining Functions

How do you define and call a Vim function, and what is the naming rule?

Define with function Name(args) … endfunction and call with call Name(). User-defined function names must begin with a capital letter.

Avoiding Flicker With Colors Name

How do you avoid re-applying (and flickering) a color scheme that hasn't changed?

Check the global colors_name variable (set by :colorscheme) and only switch when it differs: if g:colors_name !~ colorScheme | execute "colorscheme " . colorScheme | endif.

Defining Arrays

How do you define an array in Vim script and access its third element?

A comma-separated list in square brackets: let g:Favcolorschemes = ["darkblue", "morning", "shine", "evening"]. Elements index from zero, so the third element is g:Favcolorschemes[2] ("shine").

Integer Division For Buckets

How does strftime("%H") / 6 help index a four-element color-scheme array?

Integer division truncates the 0–23 hour into one of four buckets (0, 1, 2, or 3), which indexes directly into the array of four schemes, replacing a long if/elseif chain.

Redraw Command

What does the redraw statement do in a script?

It forces Vim to repaint the screen immediately, so changes (like a new color scheme) appear at once.
