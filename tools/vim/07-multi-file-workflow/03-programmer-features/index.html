What Is Folding

What does folding do in Vim, and what are the two stages?

Folding hides a range of lines and replaces them with a one-line placeholder so you can see structure instead of detail. Stage one: define what makes a fold (the fold method); stage two: open and close the folds.

Six Fold Methods

Name the six values of the foldmethod option.

manual (your own commands, the default), indent (follows indentation/shiftwidth), syntax (follows language syntax), expr (regex/expression defines folds), marker (markers in the file), and diff (differences between files).

Fold Commands Start Letter

Cloze: All fold commands begin with the letter {{z}}, because a folded sheet of paper viewed from the side looks like it.

z

Open Close Toggle Fold

Which commands open, close, and toggle a single fold?

zo opens one fold, zc closes one fold, and za toggles one fold. The uppercase versions (zO, zC, zA) act recursively on nested folds.

Delete Fold Vs Delete Text

What is the dangerous difference between zd and dd on a folded line?

zd deletes (undefines) the fold but leaves the text intact. dd on a folded line deletes ALL the lines hidden in the fold. Never confuse "delete fold" with "delete text".

Fold A Brace Block

With the cursor on a brace, how do you fold the whole { … } block?

zf% — zf starts a fold and % jumps to the matching brace, so the fold covers the entire block.

Fold Column And Numbers

Which options add visual fold cues and line numbers?

:set foldcolumn=n shows an n-column margin (max 12) where +/- mark fold starts and | marks fold bodies; :set number shows line numbers and Vim reports how many lines are hidden.

Outlining With Foldlevel

After :set foldmethod=indent, how do you show only top-level headings versus more detail?

:set foldlevel=N shows only lines whose fold level is ≤ N. A low value (e.g. 0) shows headings; raise it (zr) to reveal detail or lower it (zm) to collapse.

Four Indent Methods

List Vim's four indentation methods in order of increasing power.

autoindent (inherits previous line's indent), smartindent (autoindent plus basic C awareness), cindent (full C-syntax indenting), and indentexpr (your own expression, the most flexible).

Cindent Options

Which three options configure cindent behavior?

cinkeys (keys that trigger reindentation), cinoptions (the indentation style), and cinwords (keywords like if/else/while that add an extra indent on the next line).

Insert Mode Indent Keys

In insert mode with autoindent on, which keys add and remove an indent level?

CTRL-T adds one level of indentation and CTRL-D removes one level.

Shift Lines

How do >> and << work, and how do you shift five lines right?

>> shifts a line right and << shifts it left by shiftwidth (default 8). Prefix with a count: 5>> shifts five lines right.

Paste Option Caveat

Why set :set paste before pasting code into an auto-indented buffer?

Without it, auto-indent treats pasted text as typed input and progressively skews it rightward. :set paste disables Vim's automatic features so text comes in intact; reset with :set nopaste.

Completion Command Prefix

Cloze: Except for the generic CTRL-N, all insert-mode completion commands begin with {{CTRL-X}} followed by a second key.

CTRL-X

Cycling Completion Candidates

After triggering completion, how do you cycle candidates and accept or cancel?

CTRL-N moves forward and CTRL-P moves backward through candidates (or repeat the second key). Press ENTER to accept a match, CTRL-E to cancel.

Common Completion Methods

What do these completion commands complete from: CTRL-X CTRL-L, CTRL-X CTRL-N, CTRL-X CTRL-K, CTRL-X CTRL-F?

CTRL-X CTRL-L   Whole lines in the current file
CTRL-X CTRL-N   Keywords in the current file
CTRL-X CTRL-K   Keywords from 'dictionary' files
CTRL-X CTRL-F   Filenames

Generic Completion And Complete Option

What does CTRL-N (generic completion) do, and where are its sources defined?

It combines all the other searches into one, with sources defined in the complete option (e.g. . current buffer, w other windows, b loaded buffers, k dictionary, s thesaurus, i included files, t/] tags).

Jump To Tag Definition

Which keystroke jumps from the identifier under the cursor to its definition?

CTRL-] jumps to the definition of the tag under the cursor (using the tags file generated by ctags).

Tag Stack Navigation

What is the tag stack, and how do you go back after jumping into definitions?

The tag stack records every location you visit via tag jumps. Use CTRL-T or :pop to pop a position off the stack and return to where you came from. :tags displays the stack.

Tag Jump In Split Window

How do you split the window while following a tag?

:stag tag splits the window and follows the named tag, or CTRL-W ] splits and jumps to the tag under the cursor.

Enable Syntax Highlighting

How do you turn on syntax highlighting and force a file type when Vim can't guess?

:syntax on enables highlighting; :set syntax=sh forces a specific file type (e.g. for a new extensionless shell script).

Syntax Groups

Name some high-level syntax groups Vim uses for highlighting.

Comment, Constant, Identifier (variables and function names), Type (declarations like int/struct), and Special (delimiters). Subgroups inherit their parent group's definitions.

Color Scheme And Highlight Commands

Which command loads a color scheme, and which changes one group's colors?

:colorscheme name loads a scheme (tab-complete to cycle). :highlight group ... changes a group's colors, e.g. :highlight identifiers guifg=red. Bare :highlight lists all definitions.

Compile With Make

How do you compile from within Vim and view the errors?

:make [target] runs make (via makeprg) and captures output in the special Quickfix List window. If it doesn't open automatically, use :copen.

Navigate Quickfix Errors

Which commands move to the next and previous errors in the Quickfix list?

:cnext (:cn) jumps to the next error and :cprevious (:cp) to the previous one. You can also press ENTER on an error line in the Quickfix window. :colder/:cnewer load older/newer error lists.

Vimgrep

What does :vimgrep do, and what is its basic syntax?

:vimgrep is a built-in grep that fills the Quickfix window with matches across files, useful for refactoring. Syntax: :vimgrep[!] /pattern/[g][j] file(s) — e.g. :vimgrep /++vim++/ *.asciidoc.
