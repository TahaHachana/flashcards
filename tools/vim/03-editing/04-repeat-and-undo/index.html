The Repeat Command

What does . (the dot command) do?

It repeats the last change at the cursor's current position. It replays edits (insert/delete/change), not pure motions or searches.

Undo Last Command

What does u do, and must the cursor be on the edited line?

u undoes the last command. The cursor does not need to be on the line where the edit was made.

Uppercase U In vi

In original vi, what does U do?

U undoes all edits made on a single line, as long as the cursor has not left that line.

Redo In Vim

How do you redo (reverse an undo) in Vim, and how deep does Vim's undo go?

CTRL-R redoes. Vim supports "infinite" / multi-level undo, so you can step back through many changes and redo forward.

Undo And The Operation Boundary

After dw dw versus 2dw, what does a single u restore in each case?

dw then dw is two operations, so u restores only the last word. 2dw is one operation, so u restores both words.

Repeat A Deletion (Cloze)

Cloze: After dd deletes a line, pressing _____ deletes another line by repeating the change.

The Spot-Edit Loop

Describe the search-then-dot loop for changing a word in many scattered places.

/oldname to find the first match, cwNewname then ESC to make the change, then repeat n (next match) and . (replay the change) across the file. Press n alone to skip a match.

Why The Dot Loop Is Safe

Why might the n / . loop be preferable to a global :%s substitution?

It lets you confirm each occurrence before changing it and skip the ones that shouldn't change — useful when only some matches should be edited.

Cycling Numbered Registers

What does the sequence "1pu.u.u accomplish?

It cycles through the numbered deletion registers: "1p puts the latest deletion, u undoes it, and each . after an undo auto-increments to the next register's contents — letting you hunt for the deletion you want.

Dot Limitations

Does . repeat cursor motions and searches?

No. . only repeats the last change (text-modifying command); it does not repeat motions or searches.
