Run A Shell Command From Vim

What command runs a Unix command from inside Vim, shows its output, and returns you to editing without changing the buffer?

:!command — the ! tells ex to start a shell and run the rest of the line. Press ENTER afterward to return to the same spot.

Display Versus Insert

What is the key difference between :!cmd and :r !cmd?

:!cmd only DISPLAYS the command's output (buffer unchanged); :r !cmd READS the output INTO the buffer.

Check The Working Directory

How do you print the current working directory without leaving Vim?

:!pwd

Repeat The Last Shell Command

Cloze: To repeat the last :! command, type {{:!!}}.

:!!

Read Command Output Into Buffer

What command reads the system date/time into the buffer, and where does it land by default?

:r !date (full form :read !date). By default the output is placed after the current line.

Read Output At The Top Of File

How do you place the output of :r !cmd at the very top of the file?

Precede it with line address 0: :0r !cmd. A line address before :r controls where the output is inserted.

The % Current-Filename Token

In a shell command run from Vim, what does % expand to, as in :$r !spell %?

% expands to the current filename. :$r !spell % reads the spell-check output of the current file in at the end of the buffer.

The Spell-Check Idiom

Describe the :w then :$r !spell % workflow for fixing misspellings.

:w saves the file, then :$r !spell % appends a list of misspelled words to the bottom. You search through the file fixing each error and delete it from the list when done.

Open A Full Subshell

What command opens a full interactive subshell from Vim, and how do you return to the editor?

:sh opens a subshell. Return to Vim with CTRL-D (or type exit). It works even from gvim.

Suspend And Resume Vim

What does CTRL-Z do, and how do you get back into Vim?

CTRL-Z suspends Vim to the shell that launched it (job control). Resume with the shell command fg. In gvim it iconifies the window instead.

sh Versus CTRL-Z

What is the difference between :sh and CTRL-Z?

:sh starts a NEW subshell as a child of Vim (return with CTRL-D); CTRL-Z SUSPENDS the existing Vim process back to its parent shell (return with fg).

autowrite And Shelling Out

If the autowrite option is set, what happens before Vim runs a :! shell command?

Vim automatically writes out the changed buffer (it also does this before :n to move to the next file), protecting unsaved work.

Read-A-File Workflow

You want to read a file in but forget its name. What two commands solve this quickly?

:!ls to list the directory and spot the name, then :r newfile to read that file in after the current line.
