ex Filter Syntax

What is the general syntax for filtering a range of lines through a shell command from the ex (:) prompt?

:range!command — a line address or range, then !, then the shell command. The command's output replaces the addressed lines.

ex vs vi Filtering

What is the one essential difference between filtering text in ex mode versus vi mode?

The mechanism is identical, but in ex you describe the block with line addresses, while in vi you describe it with motions/text objects.

Sort A Specific Line Range

What command sorts lines 96 through 99 of the current file in place?

:96,99!sort

Filter The Whole File

Cloze: To filter the entire file through sort, use {{:%!sort}}, where % means {{the whole file (line 1 through last line)}}.

:%!sort; % is shorthand for 1,$, the whole file.

Filter Between Marks

What command filters from mark a to mark b through an awk program?

:'a,'b!awk '{...}'

Number The First Ten Lines

What command runs lines 1 through 10 through nl to number them?

:1,10!nl

ex Range Address Tokens

What do these ex address tokens mean: ., $, %, 'x?

.     the current line
$     the last line of the file
%     the whole file (1,$)
'x    the line holding mark x

Filter Current Line Plus Next Four

What ex command filters the current line plus the next four lines through sort?

:.,.+4!sort

Pipes Inside A Filter Command

Can you use shell pipes inside a Vim filter command?

Yes. Pipes work normally, so you can chain Unix tools, e.g. :%!awk '...' | sort -n replaces the whole buffer with the chained output.

Iterating On Filters With Undo

When experimenting with successive filter commands, why press u after each run?

Because each filter replaces the text. Pressing u restores the original input so the NEXT attempt operates on the original text, not the previous (possibly wrong) result.

Equivalent Filter Forms

Match each filter form to how it addresses the block: :n,m !command, num !! command, !motion command.

:n,m !command     ex range (lines n through m)
num !! command    vi count (num lines from cursor)
!motion command   vi motion (lines covered by motion)

Reverting A Bad Filter

After saving with :w, what ex command reloads the last-saved version if a filter mangles the buffer?

:e! — it discards unsaved changes and reloads the file from disk.
