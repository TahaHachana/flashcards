What Filtering Does To The Buffer

When you filter a block of text through a Unix command in Vim, what happens to the original block?

The block is sent to the command as standard input, and the command's standard output REPLACES the original block in the buffer. The original text is gone (recoverable only with undo).

vi-Mode Filter Syntax

What is the general syntax for filtering text through a command in vi (normal) mode?

! motion command ENTER — type !, then a motion describing the block of lines, then the shell command, then press ENTER.

Filter Next Sentence

What does !)sort do?

It passes the lines covered by the next-sentence motion ) through sort, replacing them with the sorted output.

Filter To End Of Paragraph

What command filters from the cursor to the end of the current paragraph through sort?

!}sort

Filter To End Of File

What command filters from the cursor to the end of the file through fmt?

!Gfmt

The !! Current-Line Object

Cloze: To filter only the current line through a command, the special object syntax is {{!!command}}.

!!command — the second ! is a special text object meaning "the current line."

Filter Quirk — Hidden Exclamation Mark

When you type the motion to select an object for filtering, where does the ! appear and what about the object character?

The ! appears at the bottom of the screen, but the character you typed to reference the object is NOT echoed.

Filter Quirk — Multi-Line Blocks Only

Why does !w (filter a word) fail, and which motions are valid for filtering?

Filter blocks must span more than one line. Only motions that move across whole lines work — G, { }, ( ), [[ ]], +, - — so a single-line motion like w won't work unless enough are given to exceed one line.

Filter Quirk — Whole Lines Affected

If the cursor is mid-line when you filter, is only the partial text the motion covers passed to the command?

No. Entire lines are always affected — the full lines containing the start and end of the motion are filtered, not just the partial span.

Count With The Filter Operator

Give two equivalent ways to filter the next 10 lines through a command using a count.

!10+ and 10!+ — a count may precede either the ! or the object, and both forms target the next 10 lines.

Repeat The Last Filter

What is the syntax to repeat the previous filter command on a new text object?

! object ! — the trailing ! replays the last shell command you filtered through.

Pattern As A Filter Object

Besides motions, what else can you use to specify the block to filter, and how much text does it take?

A search: /pattern followed by ENTER. It takes the text from the cursor up to the pattern as input to the command.

Filtering From Visual Mode

What are the steps to filter a selection through sort using Visual mode?

Select lines with V, type !, type sort, then press ENTER. The selection is replaced by the sorted output.

Uppercasing A Block With tr

Which shell command, used as a filter, converts a block of text from lowercase to uppercase?

tr '[:lower:]' '[:upper:]'

Filter Operator Mental Model

How is ! analogous to c, d, and y?

Like c/d/y, ! is an operator that takes a motion. But instead of changing, deleting, or yanking, it pipes the covered lines through a shell command and swaps in the command's output.
