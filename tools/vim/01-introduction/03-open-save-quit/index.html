What Is The Buffer

What is the "buffer," and how does it relate to your file on disk?

The buffer is an in-memory working copy of the file. All edits happen there; the file on disk is unchanged until you write (save) the buffer.

Command To Open A File

What shell command opens a file in Vim, and what happens if you give no filename?

vim filename (or vi filename). With no filename, Vim opens an unnamed empty buffer that you name when you first write it.

Tildes On A New File

What do the tilde (~) characters down the left edge of the screen mean?

They mark lines that do not exist in the file — not even blank lines. They appear past the end of the buffer's actual content.

The Status / Prompt Line

What does the prompt line (status line) at the bottom of the screen show when you open a file?

The file's name and status, e.g. "practice" [New file] for a brand-new file.

ZZ Command

What does ZZ do, and what must be true to use it?

In command mode, ZZ (both capital) writes the file if it changed and quits. It is the quick save-and-exit command.

Core ex Save/Quit Commands

Give the meaning of :w, :q, and :wq.

:w    Write (save), do not quit
:q    Quit (only if no unsaved changes)
:wq   Write the file, then quit

wq Versus ZZ Versus x

How do :wq, ZZ, and :x differ in when they write?

:wq writes unconditionally (always updating the modification time). ZZ and :x write only if the file was actually changed. The timestamp difference matters to tools like make.

Quit Without Saving

How do you quit and discard all edits made since the last write?

:q! — the exclamation point overrides the editor's protection and throws away unsaved changes.

Revert To Last Saved Version

Which command reloads the last saved version of the file so you can start over (while staying in the editor)?

:e! — it reloads the file from disk, discarding buffer edits since the last write.

No Write Since Last Change

What message appears if you try to :q with unsaved changes, and how do you proceed if you want to discard them?

"E37: No write since last change (add ! to override)". Use :q! to quit anyway, discarding the changes.

Save To A New Filename

How do you save the buffer to a different file while leaving the original unchanged?

:w newfile — writes the buffer's contents into newfile; the original file stays as it was. Useful for permission issues or keeping both versions.

The :w Habit

What good habit protects your edits against crashes or a bad edit during a session?

Use :w frequently throughout the session to write (save) your work.
