Defining Functions

How do you define a function in Rust?

Use the fn keyword followed by name, parameters in parentheses, and body in curly braces.

fn greet() {
    println!("Hello!");
}
greet();  // Call the function

Function Naming Convention

What naming convention does Rust use for functions?

Snake_case: lowercase with underscores separating words.

fn calculate_total() { }
fn send_email() { }
fn is_valid_user() { }

The main Function

What is special about the main function?

It's the entry point where program execution begins. Every Rust program must have a main function. It takes no parameters and returns nothing (unit type).

fn main() {
    println!("Program starts here");
}

Function Parameters

How do you define function parameters in Rust?

Use name: type syntax. Type annotations are required for parameters.

fn greet(name: &str) {
    println!("Hello, {}!", name);
}
greet("Alice");

Multiple Parameters

How do you define a function with multiple parameters?

Separate parameters with commas, each with its type annotation.

fn add(x: i32, y: i32) {
    println!("{}", x + y);
}
add(5, 3);

Parameters vs Arguments

What is the difference between parameters and arguments?

Parameters are variables in the function definition. Arguments are actual values passed when calling the function.

fn multiply(x: i32, y: i32) { }  // x, y are parameters
multiply(4, 5);  // 4, 5 are arguments

Return Values Syntax

How do you specify a function's return type?

Use an arrow -> followed by the type, placed after parameters and before the opening brace.

fn add(x: i32, y: i32) -> i32 {
    x + y
}

Implicit Returns

What is an implicit return in Rust?

The last expression in a function (without a semicolon) is automatically returned.

fn square(x: i32) -> i32 {
    x * x  // No semicolon, no return keyword needed
}

Explicit Returns

When and how do you use explicit returns?

Use the return keyword for early returns from a function.

fn absolute_value(x: i32) -> i32 {
    if x < 0 {
        return -x;  // Early return
    }
    x  // Implicit return
}

Returning Tuples

How do you return multiple values from a function?

Return a tuple containing multiple values.

fn calculate(x: i32) -> (i32, i32, i32) {
    (x + 1, x * 2, x * x)
}
let (inc, dbl, sqr) = calculate(5);

Unit Return Type

What is the implicit return type for functions that don't return a value?

The unit type (), though it's usually omitted from the signature.

fn print_msg(msg: &str) {
    println!("{}", msg);
}  // Implicitly returns ()

// Equivalent to:
fn print_msg(msg: &str) -> () { }

What Are Statements

What are statements in Rust?

Instructions that perform an action but do not return a value. They end with a semicolon.

let x = 5;  // Statement: variable binding

What Are Expressions

What are expressions in Rust?

Code that evaluates to a value. They do not end with a semicolon when used as return values.

5           // Expression: evaluates to 5
5 + 6       // Expression: evaluates to 11
x * 2       // Expression

Statements Don't Return Values

Can you assign a statement to a variable?

No. Statements don't have values, so you can't assign them.

let x = (let y = 6);  // ❌ Error: let is a statement

Expression Examples

What are some examples of expressions in Rust?

Math operations (5 + 6), function calls (add(3, 4)), blocks ({ y + 1 }), if expressions, and match expressions all evaluate to values.

let x = {
    let y = 3;
    y + 1  // Expression: block evaluates to 4
};

Semicolon Impact on Returns

What does adding a semicolon do to an expression?

It turns an expression into a statement, changing the return value to unit ().

fn returns_five() -> i32 {
    5  // Expression: returns 5
}
fn returns_nothing() {
    5;  // Statement: returns ()
}

Block Expressions

What value does a block expression evaluate to?

Blocks evaluate to their last expression (if it doesn't have a semicolon).

let x = {
    let price = 10;
    let quantity = 3;
    price * quantity  // Block evaluates to 30
};

Common Semicolon Error

What's wrong with this function and how do you fix it?

fn add(x: i32, y: i32) -> i32 {
    x + y;
}

The semicolon makes it return () instead of i32. Remove the semicolon to return the value.

fn add(x: i32, y: i32) -> i32 {
    x + y  // ✅ Now returns i32
}

Function Scope

Are variables declared inside a function accessible outside it?

No. Variables declared inside functions are local to that function.

fn example() {
    let x = 5;
}
fn main() {
    println!("{}", x);  // ❌ Error: x doesn't exist here
}

Call by Value

What happens when you pass a Copy type to a function?

The value is copied. Modifications inside the function don't affect the original.

fn modify(mut x: i32) {
    x = 10;  // Only modifies local copy
}
let num = 5;
modify(num);
println!("{}", num);  // Still 5

Call by Reference

How do you allow a function to modify the original value?

Pass a mutable reference using &mut.

fn increment(x: &mut i32) {
    *x += 1;
}
let mut num = 5;
increment(&mut num);
println!("{}", num);  // 6

Diverging Functions

What is a diverging function and how do you declare it?

A function that never returns, using the ! (never) type. Used for functions that panic or loop forever.

fn exit_program() -> ! {
    panic!("Terminated!");
}
fn infinite_loop() -> ! {
    loop { }
}

Function Pointers

Can functions be stored in variables and passed as arguments?

Yes. Use function pointers with type fn(params) -> return_type.

fn add(a: i32, b: i32) -> i32 { a + b }
fn apply(a: i32, b: i32, op: fn(i32, i32) -> i32) -> i32 {
    op(a, b)
}
let result = apply(5, 3, add);  // 8

Guard Clauses

What are guard clauses and when should you use them?

Early returns that handle edge cases or invalid inputs before main logic.

fn divide(a: f64, b: f64) -> Option<f64> {
    if b == 0.0 {
        return None;  // Guard clause
    }
    Some(a / b)
}

Missing Parameter Types Error

Can Rust infer parameter types like it infers variable types?

No. Parameter types must be explicitly annotated.

fn greet(name) { }  // ❌ Error: must specify type
fn greet(name: &str) { }  // ✅ Correct

If as Expression

Can if statements be used as expressions?

Yes. if/else can return values and be used in assignments.

let number = if condition { 5 } else { 6 };

Match as Expression

Can match statements be used as expressions?

Yes. match evaluates to a value and can be assigned.

let result = match value {
    1 => "one",
    2 => "two",
    _ => "other",
};

Function Documentation

How should you document complex functions?

Use doc comments (///) describing purpose, arguments, and behavior.

/// Calculates compound interest.
///
/// # Arguments
/// * `principal` - Initial amount
/// * `rate` - Annual rate (0.05 for 5%)
fn compound_interest(principal: f64, rate: f64) -> f64 {
    // ...
}

Function Best Practices

What are key best practices for writing functions?

Keep functions small (single purpose), use descriptive names that indicate what they do, and document complex behavior.

Nested Scope Access

Can inner scopes access variables from outer scopes?

Yes. Inner scopes can access outer scope variables, but outer scopes cannot access inner scope variables.

fn outer() {
    let x = 5;
    {
        println!("{}", x);  // ✅ Can access x
        let y = 10;
    }
    println!("{}", y);  // ❌ Error: y out of scope
}
