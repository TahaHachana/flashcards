What Are Scalar Types

What is a scalar type in Rust?

A scalar type represents a single value. Rust has four primary scalar types: integers, floating-point numbers, booleans, and characters. All are stored on the stack and implement the Copy trait.

Four Scalar Types

What are the four primary scalar types in Rust?

  1. Integers (whole numbers)
  2. Floating-point numbers (decimal numbers)
  3. Booleans (true or false)
  4. Characters (single Unicode characters)

Integer Type Categories

What are the categories of integer types in Rust?

Signed (i8, i16, i32, i64, i128, isize) and unsigned (u8, u16, u32, u64, u128, usize). Signed can be negative, unsigned can only be non-negative.

let signed: i32 = -42;    // Can be negative
let unsigned: u32 = 42;   // Must be non-negative

Signed Integer Range Formula

What is the range formula for signed integer types?

-(2^(n-1)) to 2^(n-1) - 1, where n is the number of bits. Example: i8 ranges from -128 to 127.

Unsigned Integer Range Formula

What is the range formula for unsigned integer types?

0 to 2^n - 1, where n is the number of bits. Example: u8 ranges from 0 to 255.

Architecture-Dependent Integers

What are isize and usize, and when should you use them?

Architecture-dependent integers: i32/u32 on 32-bit systems, i64/u64 on 64-bit systems. Use usize for array/vector indices, collection sizes, and memory operations.

let vec = vec![1, 2, 3];
let length: usize = vec.len();

Integer Literal Formats

What formats can integer literals use in Rust?

Decimal (98_222), hexadecimal (0xff), octal (0o77), binary (0b1111_0000), and byte (b'A' for u8 only). Underscores improve readability.

Integer Default Type

What is the default integer type when not specified?

i32. It's the most commonly used integer type and performs well on modern architectures.

let x = 42;  // Inferred as i32

Type Suffixes

How can you specify integer type directly in the literal?

Use type suffixes on the literal:

let x = 42u32;    // u32
let y = 100i64;   // i64
let z = 255u8;    // u8

Integer Overflow Behavior

How does Rust handle integer overflow in debug vs release mode?

Debug mode: panics on overflow. Release mode: wraps around. Use methods like wrapping_add, checked_add, saturating_add, or overflowing_add for explicit control.

let x: u8 = 255;
let y = x.checked_add(1);  // Returns None

Floating-Point Types

What are Rust's floating-point types and their sizes?

f32 (32-bit, single precision) and f64 (64-bit, double precision). Default is f64.

let x = 2.0;        // f64 (default)
let y: f32 = 3.0;   // f32

Floating-Point Default

Why is f64 the default floating-point type?

On modern CPUs, f64 is roughly the same speed as f32 but provides more precision (about 15-17 decimal digits vs 6-9 for f32).

Floating-Point Precision Issues

Why should you never compare floating-point numbers for exact equality?

Floating-point arithmetic has inherent precision limitations. Compare if values are close enough instead.

let x = 0.1 + 0.2;  // 0.30000000000000004
let y = 0.3;
println!("{}", x == y);  // false!
println!("{}", (x - y).abs() < 0.00001);  // true

Special Floating-Point Values

What special values can floating-point types represent?

INFINITY, NEG_INFINITY, and NAN (Not a Number).

let inf = 1.0 / 0.0;     // INFINITY
let neg_inf = -1.0 / 0.0; // NEG_INFINITY
let nan = 0.0 / 0.0;      // NAN

Boolean Type

What is the boolean type in Rust, its size, and possible values?

The bool type has two values: true or false. It's one byte in size.

let t = true;
let f: bool = false;

Boolean Operations

What are the basic boolean operations in Rust?

Logical AND (&&), logical OR (||), and logical NOT (!).

let and = true && false;    // false
let or = true || false;     // true
let not = !true;            // false

Short-Circuit Evaluation

How do && and || operators use short-circuit evaluation?

They stop evaluating as soon as the result is determined. && stops on false, || stops on true.

let x = false && expensive_function();  // Not called
let y = true || expensive_function();   // Not called

Character Type

What is the char type in Rust, its size, and what can it represent?

A Unicode Scalar Value, four bytes in size. Can represent any Unicode character including ASCII, emojis, and symbols.

let letter = 'A';
let emoji = 'ü¶Ä';
let chinese = '‰∏≠';

Character Literals Syntax

How do you write character literals vs string literals?

Characters use single quotes, strings use double quotes.

let c = 'A';       // char
let s = "A";       // &str
let invalid = 'AB'; // ‚ùå Error: too many characters

Unicode Escapes

How do you write Unicode characters using escape sequences?

Use \u{} with hex code or \x for ASCII hex.

let crab = '\u{1F980}';  // ü¶Ä
let letter = '\x41';     // 'A'

Special Character Escapes

What are common special character escape sequences?

\n (newline), \t (tab), \r (carriage return), \ (backslash), \' (single quote).

let newline = '\n';
let tab = '\t';

Integer Division

How does integer division work in Rust?

Integer division truncates toward zero, discarding the fractional part.

let a = 10 / 3;     // 3 (not 3.333...)
let b = -10 / 3;    // -3 (not -3.333...)
let c = 10.0 / 3.0; // 3.333... (float division)

Type Mismatch in Operations

Can you mix different numeric types in operations?

No. You must explicitly convert types using the as keyword.

let x: i32 = 5;
let y: f64 = 2.5;
let z = x + y;  // ‚ùå Error

let z = x as f64 + y;  // ‚úÖ OK

Type Conversion with as

How do you convert between numeric types?

Use the as keyword for explicit conversion. Be careful with narrowing conversions.

let integer: i32 = 10;
let float = integer as f64;
let large: i32 = 300;
let small = large as i8;  // Truncates to 44

Choosing Integer Types

What are best practices for choosing integer types?

Use i32 by default, u32 when never negative, i64/u64 for large numbers, usize for indices/sizes, and u8 for bytes.

Choosing Floating-Point Types

When should you use f32 vs f64?

Use f64 by default (better precision, similar performance). Use f32 for memory optimization in large arrays, graphics programming, or when precision isn't critical.

Comparison Operations

What comparison operators does Rust provide and what do they return?

== (equal), != (not equal), > (greater), < (less), >= (greater or equal), <= (less or equal). All return bool.

let equal = 5 == 5;        // true
let greater = 10 > 5;      // true

Scalar Types Are Copy

What happens when you assign or pass a scalar type to a function?

Scalar types implement Copy, so they're copied rather than moved. The original remains valid.

let x = 5;
let y = x;  // x is copied, both x and y are valid

Numeric Type Inference

How does Rust infer numeric types from context?

Rust infers types from usage. Integers default to i32, floats to f64.

let x = 5;      // i32
let y = 5.0;    // f64
let z = 42u8;   // Explicitly u8

Arithmetic Operations

What basic arithmetic operations do numeric types support?

Addition (+), subtraction (-), multiplication (*), division (/), and remainder (%).

let sum = 5 + 10;
let difference = 95 - 4;
let product = 4 * 30;
let quotient = 56 / 32;
let remainder = 43 % 5;
