What Is Control Flow

What is control flow in programming?

Control flow determines the order in which code executes. It allows you to execute code conditionally (if/else), repeat code (loops), and exit loops early (break/continue).

Basic if Expression

How do you write a basic if expression in Rust?

Use the if keyword with a boolean condition in braces.

let number = 5;
if number < 10 {
    println!("Less than 10");
}

if Condition Must Be Bool

Can you use non-boolean values as if conditions in Rust?

No. Conditions must be bool type. Rust won't automatically convert other types.

if number { }  // ❌ Error
if number != 0 { }  // ✅ OK: explicitly bool

if-else Expression

How do you provide an alternative branch in an if expression?

Use else for the alternative branch.

if number % 2 == 0 {
    println!("Even");
} else {
    println!("Odd");
}

else if Chaining

How do you chain multiple conditions in Rust?

Use else if for additional conditions. Only the first matching condition executes.

if number % 4 == 0 {
    println!("Divisible by 4");
} else if number % 2 == 0 {
    println!("Divisible by 2");
} else {
    println!("Odd");
}

if as an Expression

Can if be used in variable assignments?

Yes. if is an expression that returns a value.

let number = if condition { 5 } else { 6 };

if Expression Type Consistency

What requirement must all branches of an if expression meet when used in assignments?

All branches must return the same type.

let x = if condition { 5 } else { "six" };  // ❌ Error
let x = if condition { 5 } else { 6 };      // ✅ OK

if Without else Return Value

What does an if expression return when there's no else and the condition is false?

It returns the unit type ().

let value = if false { 5 };  // value is ()

loop Keyword

How do you create an infinite loop in Rust?

Use the loop keyword. It runs until explicitly stopped with break.

loop {
    println!("Forever!");
}

Breaking Out of Loops

How do you exit a loop early?

Use the break keyword.

let mut count = 0;
loop {
    count += 1;
    if count == 5 {
        break;
    }
}

Returning Values from loop

How can a loop return a value?

Place the value after break.

let result = loop {
    counter += 1;
    if counter == 10 {
        break counter * 2;  // Returns 20
    }
};

Loop Labels

How do you break out of an outer loop from within a nested loop?

Use loop labels starting with a single quote.

'outer: loop {
    loop {
        break 'outer;  // Breaks the outer loop
    }
}

continue Keyword

How do you skip the rest of the current iteration and start the next one?

Use the continue keyword.

for i in 1..10 {
    if i % 2 == 0 {
        continue;  // Skip even numbers
    }
    println!("{}", i);  // Only prints odd
}

while Loops

How do you create a loop that runs while a condition is true?

Use a while loop with a boolean condition.

let mut number = 3;
while number != 0 {
    println!("{}!", number);
    number -= 1;
}

while vs loop with if

What is the relationship between while loops and loop with if?

They're equivalent. while is cleaner syntax for conditional looping.

while condition { }
// Equivalent to:
loop {
    if !condition { break; }
}

for Loop Over Arrays

How do you iterate over an array using a for loop?

Use for with the array directly.

let arr = [10, 20, 30];
for element in arr {
    println!("{}", element);
}

for Loop Over Ranges

How do you use ranges in for loops?

Use .. for exclusive ranges or ..= for inclusive ranges.

for i in 1..5 {
    println!("{}", i);  // 1, 2, 3, 4
}
for i in 1..=5 {
    println!("{}", i);  // 1, 2, 3, 4, 5
}

Reverse Iteration

How do you iterate over a range in reverse?

Use .rev() on the range.

for i in (1..5).rev() {
    println!("{}", i);  // 4, 3, 2, 1
}

Iterating with Indices

How do you get both index and value when iterating?

Use enumerate() on the iterator.

let arr = [10, 20, 30];
for (index, value) in arr.iter().enumerate() {
    println!("arr[{}] = {}", index, value);
}

for vs while for Iteration

Why is for preferred over while for iteration?

for loops are safer and more concise. They eliminate bugs like forgetting to increment index or accessing out of bounds.

// Using while (error-prone)
let mut i = 0;
while i < arr.len() {
    println!("{}", arr[i]);
    i += 1;
}

// Using for (safer)
for element in arr {
    println!("{}", element);
}

When to Use loop

When should you use loop instead of while or for?

When you need an infinite loop, want to return a value from the loop, or the exit condition is complex.

When to Use while

When should you use while loops?

When you have a clear boolean condition that's checked before each iteration.

When to Use for

When should you use for loops?

When iterating over collections, iterating a specific number of times, or when you want the safest, most idiomatic code. for is generally preferred for iteration.

Exclusive vs Inclusive Ranges

What's the difference between .. and ..= in ranges?

.. is exclusive (doesn't include end), ..= is inclusive (includes end).

1..5   // 1, 2, 3, 4 (excludes 5)
1..=5  // 1, 2, 3, 4, 5 (includes 5)

break in Nested Loops

How do you break out of multiple nested loops at once?

Use labeled break with the outer loop's label.

'outer: for i in 1..=3 {
    for j in 1..=3 {
        if i == 2 && j == 2 {
            break 'outer;
        }
    }
}

Accumulator Pattern

What is the accumulator pattern in loops?

A pattern where you initialize a variable before the loop and update it in each iteration.

let mut sum = 0;
for num in numbers {
    sum += num;
}

Loop Performance

Is there a performance difference between loop, while, and for?

No. All three compile to similar machine code. Choose based on clarity and safety, not performance. for is preferred for iteration because it's safest.

Common Loop Error - Infinite Loop

What's wrong with this code?

loop {
    println!("Running...");
}

It runs forever without a break condition, so the program never exits. Add a break condition to exit the loop.

Common Loop Error - Type Mismatch

What's wrong with this if expression?

let value = if condition { 5 } else { "six" };

Type mismatch - both branches must return the same type. Fix: use the same type in both branches.

let value = if condition { 5 } else { 6 };

Best Practice - Prefer for Over while

Why should you prefer for loops over while loops for iteration?

for loops are more idiomatic, prevent common bugs (off-by-one, out-of-bounds), clearly express intent, and are more concise.

// Less idiomatic
let mut i = 0;
while i < 10 {
    println!("{}", i);
    i += 1;
}

// More idiomatic
for i in 0..10 {
    println!("{}", i);
}
