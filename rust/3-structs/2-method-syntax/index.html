Method vs Function

What distinguishes a Rust method from a free function?

A method is defined inside an impl block with its first parameter named self, and is called using dot syntax on an instance, whereas a free function is defined outside an impl and called by name.

Defining Methods

How do you define an area method for a Rectangle struct?

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

Self Parameter Types

What are the three valid types for the first parameter of a method?

  • self — takes ownership
  • &self — immutable borrow
  • &mut self — mutable borrow

Automatic Referencing and Dereferencing

How does Rust handle receiver referencing/dereferencing when calling methods?

Rust automatically inserts &, &mut, or * to make the receiver match the method signature.

p1.distance(&p2); // same as (&p1).distance(&p2)

Getters

How can you create a getter method named the same as a struct field?

Define a method with no extra parameters that returns the field or a derived value.

impl Rectangle {
    fn width(&self) -> bool {
        self.width > 0
    }
}

Methods with Parameters

How do you define a method that checks if one Rectangle can hold another?

impl Rectangle {
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}

Associated Functions

What is an associated function and how do you call one?

An associated function is defined in an impl block without a self parameter and is called with the :: syntax.

impl Rectangle {
    fn square(size: u32) -> Self {
        Self { width: size, height: size }
    }
}

let sq = Rectangle::square(3);

Multiple impl Blocks

Can you split methods for the same type across multiple impl blocks?

Yes. You can group related methods in separate impl blocks:

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

impl Rectangle {
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}
