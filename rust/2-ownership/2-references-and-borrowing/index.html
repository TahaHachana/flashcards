References Concept

What does the & operator do in Rust?

It creates a reference, allowing access to a value without taking ownership.

let s1 = String::from("hello");
let len = calculate_length(&s1); // borrow s1

Borrowing

What is borrowing in Rust?

Borrowing is using a reference to access a value without taking ownership.

  • Immutable references: read-only access.
  • Mutable references: read-write access (with restrictions).

Multiple Immutable References

Can you have multiple immutable references to the same value?

Yes. Multiple &T immutable references are allowed simultaneously because they only read the value.

let s = String::from("hello");
let r1 = &s;
let r2 = &s;
println!("{r1}, {r2}");

Mutable References Rule

What restriction applies to mutable references in Rust?

Only one mutable reference (&mut) is allowed at a time. This prevents data races at compile time.

Mutable Reference Example

How do you create and use a mutable reference in Rust?

Use &mut to borrow mutably.

fn change(s: &mut String) {
    s.push_str(", world");
}

let mut s = String::from("hello");
change(&mut s);

Data Race Conditions

What three conditions define a data race, and how does Rust prevent them?

A data race occurs when:

  1. Two or more pointers access the same data simultaneously.
  2. At least one is writing.
  3. No synchronization is used.

Rust prevents this by restricting to one mutable reference or multiple immutable ones.

Mixed References

Can mutable and immutable references coexist to the same value?

No. You cannot have a mutable reference and immutable references at the same time. They must be separated by scope.

let mut s = String::from("hello");

let r1 = &s; // immutable
let r2 = &s; // immutable
// let r3 = &mut s; // ‚ùå not allowed here

Scoped Borrowing

How can you reuse a mutable reference after another one?

Place the first borrow in a smaller scope so it ends before the next one begins.

let mut s = String::from("hello");
{
    let r1 = &mut s;
} // r1 is dropped here

let r2 = &mut s; // valid

Dangling References

Does Rust allow dangling references?

No. The compiler prevents them by enforcing lifetimes. You cannot return a reference to a value that goes out of scope.

fn dangle() -> &String { // ‚ùå error
    let s = String::from("hello");
    &s
}

Borrow Checker

What role does the borrow checker play in Rust?

The borrow checker enforces ownership and borrowing rules at compile time, ensuring:

  • No dangling references.
  • No simultaneous mutable/immutable conflicts.
  • No data races. This gives memory safety with zero runtime cost.

Borrowing Summary

Summarize the rules of references and borrowing in Rust.

  • References (&) let you access without ownership.
  • Borrowing = using references.
  • Multiple immutable borrows are allowed.
  • Only one mutable borrow at a time.
  • Mutable + immutable borrows cannot coexist.
  • Borrow checker enforces safety at compile time.
  • Dangling references are not allowed.
