Reference Definition

What is a reference in Rust and how do you write immutable and mutable references?

A reference lets you borrow data without taking ownership.
- Immutable reference: &T
- Mutable reference: &mut T 

fn calculate_length(s: &String) -> usize {
    s.len()
}

fn change(s: &mut String) {
    s.push_str(", world");
}

Immutable References

How many immutable references to the same data are allowed in a given scope?

You can have any number of immutable references (&T) simultaneously. They only permit read-only access.

let s = String::from("hello");
let r1 = &s;
let r2 = &s;
println!("{} and {}", r1, r2);

Mutable References

How many mutable references (&mut T) can you have to the same data in one scope, and why?

Only one mutable reference per scope. This ensures exclusive write access and prevents data races.

let mut s = String::from("hi");
{
    let r1 = &mut s;
    r1.push('!');
} // r1 goes out of scope here
let r2 = &mut s;
r2.push('?');
println!("{}", s); // prints "hi!?"

Borrowing Rules

What are Rust’s core borrowing rules?

  1. At most one mutable reference (&mut T) or any number of immutable references (&T) in a scope.
  2. References must always remain valid for their lifetimes.
  3. You cannot mix mutable and immutable borrows to the same data at the same time.

Dangling References

What is a dangling reference, and why does Rust forbid it?

A dangling reference points to memory that has been deallocated. Rust’s borrow checker prevents returning references to local data that goes out of scope, avoiding invalid memory access.

// ERROR: returns &String to local `s`
fn dangle() -> &String {
    let s = String::from("hello");
    &s
}
// `s` goes out of scope here, so &s is a dangling reference

Avoiding Dangling References

How do you avoid dangling references when you need to return owned data?

Return the owned value, transferring ownership to the caller instead of a reference.

fn no_dangle() -> String {
    let s = String::from("hello");
    s
}

fn main() {
    let s = no_dangle();
    println!("{}", s);
}
