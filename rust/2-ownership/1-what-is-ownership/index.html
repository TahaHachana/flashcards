Ownership Rules

What are Rust’s three ownership rules?

  1. Each value has a single owner.
  2. There can only be one owner at a time.
  3. When the owner goes out of scope, the value is dropped.

Stack vs Heap

How do stack and heap allocation differ in Rust?

  • The stack holds fixed-size values and follows LIFO (fast).
  • The heap holds dynamically sized data (slower due to allocation). 
// Stack: size known at compile time
let x: i32 = 42;

// Heap: dynamic String allocation
let s: String = String::from("hello, heap");

Move Semantics

What happens when you assign a non-Copy type (e.g., String) to another variable?

Rust moves ownership: the pointer, length, and capacity are copied, but the heap data isn’t. The original binding becomes invalid.

let s1 = String::from("hello");
let s2 = s1;       // s1 is moved → invalid
// println!("{}", s1); // ERROR: use of moved value

Deep Copy with Clone

How do you perform a deep copy of heap data in Rust?

Call .clone() to allocate new heap memory and duplicate the contents.

let s1 = String::from("hello");
let s2 = s1.clone();
println!("s1 = {}, s2 = {}", s1, s2); // both valid

The Copy Trait

What is the Copy trait and which types implement it?

Types with fixed size (no heap data) implement Copy. Assignment makes a bitwise copy; the original remains valid. 

let a = 5;     // i32 implements Copy
let b = a;     // a is still valid
println!("a = {}, b = {}", a, b);

Common Copy types: integers, booleans, floats, char, and tuples of Copy types.

Ownership in Function Calls

How does ownership transfer when passing arguments to functions?

  • Passing a non-Copy type moves ownership.
  • Passing a Copy type duplicates the value.
fn takes_ownership(s: String) { /* s dropped here */ }
fn makes_copy(x: i32)      { /* x is Copy */ }

let s = String::from("rust");
takes_ownership(s); // s moved → invalid after

let x = 42;
makes_copy(x);      // x copied → still valid
println!("{}", x);  // prints 42

Return Ownership

How can a function transfer ownership back to its caller?

By returning the value, ownership moves to the variable that captures the return.

fn gives_ownership() -> String {
    String::from("yours")
}

fn takes_and_gives_back(s: String) -> String {
    s
}

let s1 = gives_ownership();            // s1 owns returned String
let s2 = takes_and_gives_back(s1);     // s1 moved into fn and then returned

References and Borrowing

What are references and why use them?

References (&T or &mut T) allow borrowing data without taking ownership, avoiding moves and extra returns.

fn calculate_length(s: &String) -> usize {
    s.len()
}

let s = String::from("hello");
let len = calculate_length(&s);
println!("Length: {}", len); // s still valid
