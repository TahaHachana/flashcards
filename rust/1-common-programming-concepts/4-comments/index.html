Purpose Of Comments

What is the purpose of comments in Rust?

Comments explain code intent for humans.
They are ignored by the compiler and should supplement clear naming rather than replace it.

Line Comments

How do you write a single-line comment in Rust?

Start with //. Everything after // on the same line is ignored by the compiler.

// This is a comment
println!("Hello, world!");

Inline Comments

How do you write an inline comment in Rust?

Place // after code on the same line.

let x = 5; // x holds the value 5

Block Comments

Does Rust support block comments, and what makes them unique?

Yes, Rust supports block comments with /* ... */. Unlike many languages, they can be nested.

/* This is a block comment
   spanning multiple lines */

/* You can /* nest */ block comments */

Documentation Comments

What special comment syntax does Rust provide for generating documentation?

Rust provides documentation comments:

  • /// → documents the item that follows (outer doc comment).
  • //! → documents the enclosing item, like a module or crate (inner doc comment).

These integrate with rustdoc to generate API docs.

/// Adds one to the input value
fn add_one(x: i32) -> i32 {
    x + 1
}

Key Comment Rules

Summarize the key comment rules in Rust.

  • // for single-line and inline comments.
  • /* ... */ for block comments (nestable).
  • /// and //! for documentation comments with rustdoc.
  • Comments improve readability but do not affect execution.
