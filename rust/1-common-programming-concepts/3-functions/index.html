Function Declaration Basics

How do you declare a function in Rust, and what naming convention is used?

Use the fn keyword followed by the function name in snake_case, parentheses for parameters, and curly braces for the body.

Example:

fn another_function() {
    println!("Another function.");
}

Calling Functions

How do you call a function in Rust, and does its position in the source file matter?

Call a function by its name followed by parentheses. The function can be defined before or after the caller, as long as it is in scope.

Example:

fn main() {
    another_function();
}

Function Parameters

What is the difference between parameters and arguments in Rust functions?

Parameters are variables in the function signature; arguments are the concrete values passed when calling the function.

Example:

fn another_function(x: i32) {
    println!("The value of x is: {x}");
}

Multiple Parameters

How do you define multiple parameters in a Rust function?

Separate parameter declarations with commas, and specify the type for each.

Example:

fn print_labeled_measurement(value: i32, unit_label: char) {
    println!("The measurement is: {value}{unit_label}");
}

Statements vs Expressions

What is the difference between statements and expressions in Rust?

Statements perform actions and do not return values. Expressions evaluate to a value and can be part of statements.

Example:

let y = 6; // statement
5 + 6      // expression

Expressions in Blocks

How can a block return a value in Rust?

A block without a semicolon on its final line is an expression whose value is returned.

Example:

let y = {
    let x = 3;
    x + 1 // returns 4
};

Semicolon Effect

What happens if you add a semicolon to the end of an expression in Rust?

It becomes a statement and no longer returns a value.

Example:

x + 1; // statement, returns ()

Functions with Return Values

How do you specify and return a value from a function in Rust?

Declare the return type after -> and use the final expression in the function body without a semicolon.

Example:

fn five() -> i32 {
    5
}

Using Return Values

How can you use a function's return value to initialize a variable?

Assign the function call to a variable.

Example:

let x = five(); // x is 5

Return Value Type Mismatch

What error occurs if a function's body ends with a statement but a return type is declared?

A mismatched types error occurs because statements return () (unit type), not the declared type.

Example:

fn plus_one(x: i32) -> i32 {
    x + 1; // error: expected i32, found ()
}

Returning Results

How can you return a Result from a function and propagate errors?

Use Result<T, E> as the return type, and the ? operator to propagate errors.

Example:

use std::fs::File;
use std::io;

fn try_open() -> io::Result<File> {
    File::open("Cargo.toml")
}

Generics and Trait Bounds

How do you define a generic function with trait bounds?

Use angle brackets with T and specify required traits after :.

Example:

fn show<T: std::fmt::Display>(x: T) {
    println!("{x}");
}

impl Trait Syntax

What is the impl Trait syntax used for in function parameters and return types?

It provides shorthand for generics:

  • In parameters, it accepts any type implementing the trait.
  • In return values, it hides the concrete type but guarantees it implements the trait.

Example:

fn log_line(item: impl std::fmt::Display) {
    println!("{item}");
}

Methods vs Associated Functions

What is the difference between a method and an associated function?

  • A method takes self (or &self, &mut self) as its first parameter and is called on an instance.
  • An associated function does not take self and is called with the type name (often used as constructors).

Example:

impl Circle {
    fn area(&self) -> f64 { 3.14 * self.r * self.r }
    fn new(r: f64) -> Circle { Circle { r } }
}

Closure Traits

What are the three closure traits, and when are they used?

  • Fn – captures by reference, can be called multiple times.
  • FnMut – captures mutably, can modify captured values.
  • FnOnce – consumes captured values, can only be called once.

Example:

fn apply<F: Fn()>(f: F) { f(); }

Lifetime Annotations in Functions

When do you need to add explicit lifetime annotations in function signatures?

When a function returns a reference tied to one of its input references, and Rust cannot infer the relationship.

Example:

fn longest<'a>(a: &'a str, b: &'a str) -> &'a str {
    if a.len() > b.len() { a } else { b }
}

Accepting Both String and \&str

How can you write a function that accepts both String and &str?

Use a generic parameter with the AsRef<str> trait.

Example:

fn greet<S: AsRef<str>>(name: S) {
    println!("Hello, {}", name.as_ref());
}

Const Functions

What are const fns in Rust, and how are they used?

  • const fn can be evaluated at compile time in constant contexts.
  • They can also be called at runtime.
  • Useful for defining constants that need computation.

Example:

const fn square(x: i32) -> i32 { x * x }
const N: i32 = square(12);

fn main() {
    println!("N = {N}");
}
