Rust Type System Basics

What does it mean that Rust is statically typed, and when is a type annotation required?

Rust is statically typed, meaning all variable types must be known at compile time.
The compiler can usually infer types, but annotations are required when:

  • Multiple types are possible (ambiguity).
  • Using generic methods like parse().
  • You want a type other than the default (i32 for integers, f64 for floats).

Example:

let guess: u32 = "42".parse().expect("Not a number!");

Floating-Point Types

What floating-point types does Rust have, and which is the default?

Rust has:

  • f32 (32-bit, single precision)
  • f64 (64-bit, double precision, default)

Both are signed and follow IEEE-754.

Example:

let y: f32 = 3.0; // f32

Array Basics

How do you declare arrays in Rust, and what are their constraints?

  • Arrays have fixed length.
  • All elements must be the same type.
  • Stored on the stack (faster than Vec).

Accessing out-of-bounds indices causes a runtime panic.

Copy vs Move Types

Which types implement Copy in Rust, and which don‚Äôt?

Copy types: cheap to duplicate; ownership doesn‚Äôt move. Examples: integers, floats, bool, char, unit ().

Non-Copy types: ownership moves by default. Examples: String, Vec<T>, Box<T>.

Float Comparison Gotcha

Why is comparing floats with == unreliable in Rust?

Floats use IEEE-754 and can‚Äôt represent some values exactly. Direct equality may fail. Use an epsilon check:

let b = 0.3;
assert!((a - b).abs() < f64::EPSILON);

String Indexing Gotcha

Why can‚Äôt you index into a Rust String with s[0]?

Rust String is UTF-8 encoded; characters may use multiple bytes. Direct indexing is ambiguous and not allowed.

Example:

// println!("{}", s[0]); // error
println!("{}", &s[0..1]); // "h"

Arrays vs Vec

When should you use an array versus a Vec<T> in Rust?

  • Array [T; N]: fixed size known at compile time, stack-allocated, very fast.
  • Vec: dynamic size, heap-allocated, can grow/shrink at runtime.

Slices Lifetime Gotcha

Why must you be careful with slices and lifetimes?

Slices borrow the original data but don‚Äôt own it. If the owner goes out of scope, the slice becomes invalid.

Example:

{
    let arr = [1, 2, 3];
    slice = &arr[0..2]; // borrows arr
}
// println!("{:?}", slice); // error: arr dropped

When to Use Arrays and Vec

Give practical guidance on choosing between arrays and Vec<T>.

  • Use arrays for: fixed-size, small, compile-time data.
  • Use Vec for: runtime-sized collections or data that grows/shrinks.

When to Use \&str vs String

When should you use &str vs String?

  • &str: read-only, borrowed string slices; function parameters when you don‚Äôt need ownership.
  • String: owned, growable string; when you need to store or modify text.

Tuples vs Structs

When should you use a tuple versus a struct?

  • Tuple: temporary grouping of values, especially when names would be redundant.
  • Struct: when values have meaning, names improve clarity, or the data is part of a stable API.
