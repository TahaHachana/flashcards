What Problem Do Slices Solve

What problem do slices solve?

Referring to part of a collection without taking ownership. Without slices, returning indices creates separate values that can become invalid if underlying data changes. Slices stay connected to the data through the borrow checker.

What Is a Slice

What is a slice in Rust?

A dynamically-sized view into a contiguous sequence. It's a reference with two components: pointer to starting element and length. Doesn't own data, just provides a window into it.

Slices Are References

How do slices relate to references?

Slices are references that follow borrowing rules. They don't own data, can be immutable (&[T]) or mutable (&mut [T]), can't outlive data, and same mutability exclusivity rules apply.

Why Slices Matter

What are the benefits of slices?

Safety (borrow checker prevents invalidation), flexibility (work with parts without copying), efficiency (just pointer and length), and generality (same concept for strings, arrays, vectors).

Creating String Slices

How do you create a string slice?

Use range syntax in square brackets.

let s = String::from("hello world");
let hello = &s[0..5];   // "hello"
let world = &s[6..11];  // "world"

Range Syntax Patterns

What are the range syntax patterns for slices?

[start..end] (exclusive end), [..end] (from beginning), [start..] (to end), [..] (entire sequence).

&s[0..5]  // From 0 to 5
&s[..5]   // From start to 5
&s[3..]   // From 3 to end
&s[..]    // Entire string

String Slice Type

What is the type of a string slice?

&str (string slice). An immutable reference to part of a string.

let s = String::from("hello");
let slice: &str = &s[0..2];

String Literals Are Slices

What type are string literals?

&str. They're slices pointing to specific points in the binary, which is why they're immutable.

let s: &str = "Hello, world!";

Why Use &str Parameters

Why use &str instead of &String for function parameters?

More flexible. Works with String references (through deref coercion), string literals, and slices. Best practice for functions that only read strings.

fn process(s: &str) { }  // Works with all string types

String vs &str

What's the difference between String and &str?

String: owned, heap-allocated, growable, can be modified. &str: borrowed reference, immutable, points to existing string data.

let owned = String::from("hello");
let borrowed: &str = &owned;

String Slice UTF-8 Boundaries

What requirement must string slices satisfy?

Must occur at valid UTF-8 character boundaries. Rust strings are UTF-8, so each character can be multiple bytes. Slicing mid-character causes panic.

let s = String::from("你好");
let slice = &s[0..3];  // ✅ One character
let slice = &s[0..1];  // ❌ Panics: not on boundary

Array Slice Type

What is the type of an array or Vec slice?

&[T] where T is the element type.

let arr = [1, 2, 3, 4, 5];
let slice: &[i32] = &arr[1..3];  // [2, 3]

One Function Multiple Types

Why can one function with &[T] parameter work with both arrays and vectors?

&[T] is a general slice type that works with any contiguous sequence. Arrays and vectors can both be sliced into &[T].

fn sum(s: &[i32]) -> i32 { }
sum(&arr[..]);  // Works with arrays
sum(&vec[..]);  // Works with vectors

Mutable Slices

How do you create and use mutable slices?

Use &mut with range syntax. Original data must be mutable. Only one mutable slice at a time.

let mut arr = [1, 2, 3];
let slice = &mut arr[1..3];
slice[0] = 10;  // Modify through slice

Slices Borrow Data

Do slices take ownership of data?

No. Slices borrow data without taking ownership. Original data remains valid after creating a slice.

let vec = vec![1, 2, 3];
let slice = &vec[1..3];  // Borrow
// vec still valid

Cannot Modify While Sliced

Can you modify a collection while an immutable slice exists?

No. The immutable slice prevents modification.

let slice = &vec[1..4];
vec.push(6);  // ❌ Error: cannot borrow as mutable
println!("{:?}", slice);

Mutable Slice Exclusivity

Can you access the original collection while a mutable slice exists?

No. Exclusive mutable access - can't access original while mutable slice exists.

let slice = &mut vec[1..4];
println!("{:?}", vec);  // ❌ Error
slice[0] = 10;

Slices Cannot Outlive Data

Can slices outlive the data they reference?

No. Same lifetime rules as references - slices can't outlive their data.

let slice;
{
    let vec = vec![1, 2, 3];
    slice = &vec[..];  // ❌ Error: vec doesn't live long enough
}

Slice Methods

What are some useful slice methods?

len(), is_empty(), first(), last(), get() (safe access), split_at() (split into two), chunks() (iterate in chunks).

slice.len();       // Length
slice.get(2);      // Some(&value) or None
slice.split_at(2); // (left, right)

Slice Memory Layout

What does a slice contain internally?

A "fat pointer" with two components: pointer to first element and length. Size: two usize values (16 bytes on 64-bit systems).

// Internally: ptr + len
let slice = &arr[1..4];

Why Slices Are Efficient

Why is creating a slice efficient?

Just copies pointer and length, doesn't copy the data. Passing slices is cheap - only two values.

let large = vec![0; 1_000_000];
let slice = &large[..];  // Fast: just pointer + length

String to &str Conversion

How do you convert String to &str?

Multiple ways: &s, &s[..], or s.as_str().

let s = String::from("hello");
let slice: &str = &s;
let slice = &s[..];
let slice = s.as_str();

&str to String Conversion

How do you convert &str to String?

Multiple ways: to_string(), String::from(), or to_owned().

let slice = "hello";
let owned = slice.to_string();
let owned = String::from(slice);
let owned = slice.to_owned();

Vec to Slice Conversion

How do you get a slice from a Vec?

Multiple ways: &vec, &vec[..], or vec.as_slice().

let vec = vec![1, 2, 3];
let slice: &[i32] = &vec;
let slice = &vec[..];
let slice = vec.as_slice();

Prefer Slices in Parameters

Why prefer slice parameters over owned or full collection references?

More flexible and efficient. Works with arrays, vectors, and subranges. Just borrows, no allocation or deallocation.

// Less flexible
fn process(v: &Vec<i32>) { }

// More flexible
fn process(s: &[i32]) { }

Slices for Subranges

How do you pass part of a collection to a function?

Use slices with range syntax.

fn process_half(data: &[i32]) { }
let vec = vec![1, 2, 3, 4, 5, 6];
process_half(&vec[..3]);  // First half

Safe Slice Access

How do you safely slice without panicking on invalid indices?

Use get() which returns Option instead of panicking.

let slice = &s[start..end];  // Panics if invalid
let slice = s.get(start..end);  // Returns None if invalid

Iterating Over Slices

How do you iterate over a slice?

Use for loop directly or with iter() for more control.

for &num in slice {
    println!("{}", num);
}

for (i, &num) in slice.iter().enumerate() {
    println!("Index {}: {}", i, num);
}

Slices vs Owned Collections Performance

Why are slices more efficient than owned collections for read-only functions?

Slices just borrow - no allocation, copying, or deallocation. Owned collections require taking ownership and eventual freeing.

fn read(data: &[i32]) { }  // Just borrows
fn own(data: Vec<i32>) { }  // Must allocate/free

Slices Complete Ownership Story

How do slices complete Rust's ownership model?

You can own data (String, Vec), borrow all of it (&String, &Vec), or borrow part of it (&str, &[T]). This flexibility with compiler-enforced safety makes the memory model powerful and practical.
