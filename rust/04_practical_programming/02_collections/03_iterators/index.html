Three Iterator Methods Overview

What are the three core iterator methods in Rust and what does each return?

  • .iter() → Returns &T (immutable references), borrows collection
  • .iter_mut() → Returns &mut T (mutable references), borrows mutably
  • .into_iter() → Returns T (owned values), consumes collection

After .into_iter(), the collection is gone. After .iter() or .iter_mut(), it still exists.

Iter Ownership and Collection State

After calling each iterator method, can you still use the original collection?

  • .iter(): ✅ Yes, collection still exists (borrowed immutably)
  • .iter_mut(): ✅ Yes, collection still exists (borrowed mutably)
  • .into_iter(): ❌ No, collection was consumed (moved)

Example: After vec.into_iter(), you cannot use vec anymore.

For Loop Default Behavior

What does for item in vec do to ownership? How do you prevent this?

for item in vec calls .into_iter(), consuming the collection.

Prevent by borrowing:

for item in &vec { }      // Same as vec.iter()
for item in &mut vec { }  // Same as vec.iter_mut()

Use &vec to keep the collection usable afterward.

Iter Mut Dereferencing

Why do you need * when modifying values with .iter_mut()?

.iter_mut() returns &mut T (mutable references), not values. To modify the actual value, you must dereference:

for num in vec.iter_mut() {
    *num += 1;  // Dereference to modify
}

Without *, you'd try to operate on a reference, which won't work.

Iterator Next Method

What does .next() return and how does it work?

Returns Option<T>: - Some(value) if items remain - None when iterator is exhausted

let mut iter = vec![1, 2].iter();
assert_eq!(iter.next(), Some(&1));
assert_eq!(iter.next(), Some(&2));
assert_eq!(iter.next(), None);  // Stays None forever

Lazy Iterator Evaluation

What does "iterators are lazy" mean? Give an example.

Iterators don't execute until consumed. They build a plan but do nothing:

vec.iter().map(|x| x * 2);  // ⚠️ Does nothing!

// Must consume:
let result: Vec<i32> = vec.iter()
    .map(|x| x * 2)
    .collect();  // Now it executes

Compiler warns: "iterators are lazy and do nothing unless consumed"

Map Adapter Purpose

What does .map() do and is it lazy?

.map() applies a function to each item and passes it on to create a new iterator.

Yes, it's lazy - nothing happens without .collect() or similar:

let doubled: Vec<i32> = vec.iter()
    .map(|x| x * 2)  // Transform each item
    .collect();      // Actually execute

Filter Adapter Purpose

What does .filter() do and what should the closure return?

.filter() keeps only items matching a condition. Closure must return bool:

let evens: Vec<i32> = vec.iter()
    .filter(|&&x| x % 2 == 0)  // Keep if true
    .cloned()
    .collect();

Returns true to keep item, false to discard.

Collect Consumer Purpose

What does .collect() do and why does it often need type annotations?

.collect() consumes an iterator and gathers items into a collection (Vec, HashMap, etc.).

Needs type annotation because it can build many types:

let vec: Vec<i32> = iter.collect();
// or
let vec = iter.collect::<Vec<i32>>();

Enumerate Adapter

What does .enumerate() do and what type does it return?

Adds an index to each item, returning (usize, T) tuples:

for (index, value) in vec.iter().enumerate() {
    println!("Index {}: {}", index, value);
}
// Output: Index 0: ..., Index 1: ..., etc.

Index starts at 0.

Skip and Take Adapters

What do .skip(n) and .take(n) do? Give an example.

  • .skip(n): Skip first n items
  • .take(n): Take next n items
let result: Vec<i32> = vec![0,1,2,3,4,5,6,7]
    .iter()
    .skip(3)   // Skip [0,1,2]
    .take(4)   // Take [3,4,5,6]
    .cloned()
    .collect();
// result = [3, 4, 5, 6]

For Each Consumer

What does .for_each() do and when should you use it instead of .collect()?

.for_each() executes a closure on each item without collecting results. Use for side effects:

vec.iter().for_each(|x| println!("{}", x));

// Equivalent to:
for x in vec.iter() {
    println!("{}", x);
}

Use when you don't need a new collection, just want to do something with each item.

Fold Aggregation

What does .fold() do and what are its two parameters?

.fold() combines all items into a single value (reduce/aggregate).

Parameters: 1. Initial accumulator value 2. Closure: |accumulator, item| -> new_accumulator

let sum = vec.iter().fold(0, |acc, x| acc + x);
// 0 + 1 + 2 + 3 = 6

Chaining Adapters

Show how to chain multiple iterator adapters and explain why it's efficient.

let result: Vec<i32> = vec.iter()
    .filter(|&&x| x > 5)
    .map(|x| x * 2)
    .take(3)
    .collect();

Efficient due to zero-cost abstractions - compiler optimizes the chain to be as fast as hand-written loops. No intermediate collections created.

Zero Cost Abstractions

What are "zero-cost abstractions" in the context of iterators?

High-level iterator code compiles to the same efficiency as low-level loops:

// This elegant code:
vec.iter().map(|x| x * 2).collect()

// Compiles as fast as:
let mut result = Vec::new();
for x in &vec {
    result.push(x * 2);
}

No performance penalty for abstraction!

Double Reference in Filter

Why do you often need |&&x| in .filter() when using .iter()?

.iter() gives &T, so the closure parameter is &&T:

vec.iter()
    .filter(|&&x| x > 5)  // x is &&i32

First & is from closure parameter, second & is from .iter().

Alternative: |x| **x > 5 (double dereference)

Cloned Adapter

What does .cloned() do and when is it commonly used?

Converts &T to T by cloning (requires Clone trait):

let vec = vec![1, 2, 3];
let owned: Vec<i32> = vec.iter()  // &i32
    .cloned()                      // i32
    .collect();

Common after .filter() to go from references to owned values.

Iterator Consumption Warning

What does this compiler warning mean and how do you fix it? "warning: unused Map that must be used; iterators are lazy"

You created an iterator chain but didn't consume it:

// ❌ Warning:
vec.iter().map(|x| x * 2);

// ✅ Fixed - consume with collect:
let result: Vec<i32> = vec.iter()
    .map(|x| x * 2)
    .collect();

Must call a consuming method like .collect(), .for_each(), etc.

Zip Adapter

What does .zip() do? Show an example creating a HashMap.

Combines two iterators into tuples:

use std::collections::HashMap;

let keys = vec!["a", "b", "c"];
let values = vec![1, 2, 3];

let map: HashMap<_, _> = keys.into_iter()
    .zip(values.into_iter())
    .collect();
// {"a": 1, "b": 2, "c": 3}

Count Consumer

What does .count() do and give an example with filtering.

Consumes iterator and returns the number of items:

let count = vec![1, 2, 3, 4, 5]
    .iter()
    .filter(|&&x| x > 2)
    .count();
// count = 3 (elements 3, 4, 5)

Useful after filtering to count matching items.

Sum Consumer

How do you sum all elements in a vector using iterators?

let sum: i32 = vec![1, 2, 3, 4, 5]
    .iter()
    .sum();
// sum = 15

// Or with fold:
let sum = vec.iter().fold(0, |acc, x| acc + x);

Type annotation often needed for .sum().

Iter vs Into Iter Type Difference

What's the difference in closure parameter types between .iter() and .into_iter()?

// .iter() - parameter is &T
vec.iter().for_each(|x| {
    // x is &i32
});

// .into_iter() - parameter is T
vec.into_iter().for_each(|x| {
    // x is i32 (owned)
});

Find Method

What does .find() return and how is it different from .filter()?

.find() returns Option<T> with the first matching item, then stops:

let result = vec.iter().find(|&&x| x > 5);
// Some(&6) if found, None if not

.filter() returns an iterator of all matching items:

let results: Vec<i32> = vec.iter()
    .filter(|&&x| x > 5)
    .cloned()
    .collect();

Any and All Methods

What do .any() and .all() do?

Both take a predicate and return bool:

.any(): True if at least one item matches

let has_even = vec![1, 3, 4].iter().any(|&x| x % 2 == 0);
// true (4 is even)

.all(): True if all items match

let all_positive = vec![1, 2, 3].iter().all(|&x| x > 0);
// true

Position Method

What does .position() return and how is it different from .find()?

.position() returns Option<usize> with the index of first match:

let pos = vec![10, 20, 30].iter()
    .position(|&x| x == 20);
// Some(1)

.find() returns Option<&T> with the actual value:

let val = vec.iter().find(|&&x| x == 20);
// Some(&20)

Rev Adapter

What does .rev() do?

Reverses the iterator direction:

let vec = vec![1, 2, 3];
let reversed: Vec<i32> = vec.iter()
    .rev()
    .cloned()
    .collect();
// [3, 2, 1]

// Useful for efficiency:
vec.iter().rev().find(|&&x| x == 5)
// Searches from end, faster if item is near end

Cycle Adapter

What does .cycle() create and when should you use it carefully?

Creates an infinite iterator that repeats forever:

let cycle = ["a", "b"].iter().cycle();
// a, b, a, b, a, b, ... forever

// Use with .take() to limit:
let limited: Vec<&str> = cycle.take(5).collect();
// ["a", "b", "a", "b", "a"]

Warning: Without .take() or similar, will loop forever!

Collect Type Flexibility

Show three different ways to specify the type when using .collect().

// 1. Variable type annotation
let vec: Vec<i32> = iter.collect();

// 2. Turbofish syntax
let vec = iter.collect::<Vec<i32>>();

// 3. Type inference with underscore
let vec: Vec<_> = iter.collect();

Iter Mut Modify Pattern

Show how to double all values in a vector in place using iterators.

let mut vec = vec![1, 2, 3, 4, 5];

vec.iter_mut().for_each(|x| *x *= 2);
// vec is now [2, 4, 6, 8, 10]

// Or with for loop:
for x in vec.iter_mut() {
    *x *= 2;
}

Must use .iter_mut() and dereference with *.

Filter Map Adapter

What does .filter_map() do and what should the closure return?

Combines filter and map - closure returns Option<T>: - Some(value): Keep and map to value - None: Filter out

let strings = vec!["1", "two", "3"];
let numbers: Vec<i32> = strings.iter()
    .filter_map(|s| s.parse().ok())
    .collect();
// [1, 3] - "two" filtered out

Peekable Iterator

What does .peekable() enable and how do you use it?

Allows looking at next item without consuming it:

let mut iter = vec![1, 2, 3].into_iter().peekable();

if let Some(&next) = iter.peek() {
    println!("Next is: {}", next);  // 1
}
let first = iter.next();  // Still Some(1)

.peek() returns Option<&T> without advancing iterator.

Flat Map Adapter

What does .flat_map() do? Give an example.

Maps each item to an iterator, then flattens all results:

let nested = vec![vec![1, 2], vec![3, 4]];

let flat: Vec<i32> = nested.iter()
    .flat_map(|inner| inner.iter())
    .cloned()
    .collect();
// [1, 2, 3, 4]

Useful for nested structures.

Partition Method

What does .partition() do?

Splits into two collections based on predicate:

let vec = vec![1, 2, 3, 4, 5, 6];

let (evens, odds): (Vec<i32>, Vec<i32>) = vec
    .into_iter()
    .partition(|&x| x % 2 == 0);

// evens = [2, 4, 6]
// odds = [1, 3, 5]

Must specify both tuple types.

Take While Adapter

What does .take_while() do and how is it different from .take()?

.take_while(): Takes items while predicate is true, stops at first false:

let result: Vec<i32> = vec![1, 2, 3, 4, 5]
    .into_iter()
    .take_while(|&x| x < 4)
    .collect();
// [1, 2, 3] - stops at 4

.take(n): Takes exactly n items regardless of values.

Skip While Adapter

What does .skip_while() do?

Skips items while predicate is true, then yields rest:

let result: Vec<i32> = vec![1, 2, 3, 4, 5]
    .into_iter()
    .skip_while(|&x| x < 3)
    .collect();
// [3, 4, 5] - skipped [1, 2]

Stops skipping at first false, returns everything after.

Iterator Performance

Why are Rust iterators as fast as manual loops?

Zero-cost abstractions: Iterator chains are optimized at compile time to eliminate: - Intermediate allocations - Unnecessary copies - Function call overhead

The high-level code compiles to same machine code as hand-written loops.

Max and Min Methods

What do .max() and .min() return?

Return Option<T> with maximum/minimum value:

let max = vec![1, 5, 3, 9, 2].into_iter().max();
// Some(9)

let min = vec![1, 5, 3, 9, 2].into_iter().min();
// Some(1)

let empty: Vec<i32> = vec![];
let max = empty.iter().max();
// None

Inspect Adapter

What does .inspect() do and when is it useful?

Calls a closure on each item for debugging/logging without consuming items:

let result: Vec<i32> = vec![1, 2, 3]
    .into_iter()
    .inspect(|x| println!("Before: {}", x))
    .map(|x| x * 2)
    .inspect(|x| println!("After: {}", x))
    .collect();

Useful for debugging iterator chains.

Collect Into HashMap

How do you collect an iterator of tuples into a HashMap?

use std::collections::HashMap;

let pairs = vec![("a", 1), ("b", 2), ("c", 3)];

let map: HashMap<_, _> = pairs
    .into_iter()
    .collect();

// Or from separate iterators:
let map: HashMap<_, _> = keys.into_iter()
    .zip(values)
    .collect();

Iterator Chain Method

What does .chain() do?

Connects two iterators into one sequential iterator:

let vec1 = vec![1, 2, 3];
let vec2 = vec![4, 5, 6];

let combined: Vec<i32> = vec1.iter()
    .chain(vec2.iter())
    .cloned()
    .collect();
// [1, 2, 3, 4, 5, 6]

First iterator exhausted, then second.
