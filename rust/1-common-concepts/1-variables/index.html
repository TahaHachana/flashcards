Variable Default State

What is the default state of variables in Rust?

Variables in Rust are immutable by default. This means once a value is bound to a variable, you cannot change that value unless explicitly made mutable.

Make Variable Mutable

How do you make a variable mutable in Rust?

Use the mut keyword: let mut x = 5;

Reassign Immutable Variable

What happens if you try to reassign an immutable variable?

You get a compile-time error. For example:

let x = 5;
x = 6; // Error: cannot assign twice to immutable variable

Variables vs Constants

What are the key differences between variables and constants in Rust?

  • Constants are always immutable (no mut allowed)
  • Constants must have type annotations
  • Constants must be set to compile-time constant expressions
  • Constants can be declared in any scope, including global

What is Shadowing

What is shadowing in Rust?

Shadowing allows you to redeclare a variable with the same name using let, which creates a new variable that shadows the previous one.

Shadowing Type Transformation

Give an example of shadowing with type transformation.

let spaces = "   ";           // &str type
let spaces = spaces.len();    // usize type

Shadowing vs Mutating

What's the difference between shadowing and mutating a variable?

  • Shadowing: Creates a new variable (can change type): let x = "hello"; let x = x.len();
  • Mutating: Changes existing variable's value (same type): let mut x = 5; x = 6;

Change Mutable Variable Type

Can you change the type of a mutable variable?

No! This will cause a compile error:

let mut spaces = "   ";
spaces = spaces.len(); // Error: mismatched types

Immutable by Default Reason

Why does Rust make variables immutable by default?

It promotes safety and makes code easier to reason about. It prevents accidental mutations and helps catch bugs at compile time.

Valid Constant Declaration

What's a valid constant declaration in Rust?

const MAX_POINTS: u32 = 100_000;
  • Must use const keyword
  • Must have type annotation
  • Must be compile-time constant expression

Multiple Shadowing

Can you shadow a variable multiple times?

Yes! Each let creates a new variable:

let x = 5;
let x = x + 1;    // x is now 6
let x = x * 2;    // x is now 12

Shadowing Benefits

What are the benefits of shadowing over using mutable variables?

  • Keeps variables immutable after transformations
  • Allows type changes
  • Makes intent clearer for one-time transformations
  • Prevents accidental mutations later in code
